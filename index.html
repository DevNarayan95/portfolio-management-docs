Core Models

  • User: Authentication, profiles, and account management with soft delete support
  • Portfolio: Investment collections with metadata and organizational structure
  • Investment: Individual assets (stocks, mutual funds, bonds, crypto) with SIP support and transaction tracking
  • Transaction: Complete buy/sell history with audit trail, linked to both investments and portfolios
  • RefreshToken: Secure JWT token lifecycle management with expiration tracking

Enums

  • InvestmentType: MUTUAL_FUND, STOCK, BOND, CRYPTOCURRENCY
  • TransactionType: BUY, SELL

Key Features

  • Soft Deletes: User, Portfolio, and Investment models support soft deletion
  • Cascade Deletes: Deleting a user removes all portfolios, investments, and tokens
  • SIP Support: Investments can track Systematic Investment Plans with amount, start date, and duration
  • Dual Tracking: Transactions reference both their investment and parent portfolio for flexible querying
  • Indexing: Optimized queries with indexes on foreign keys and email

Delete portfolio (soft delete)

Investment Endpoints

GET /portfolios/:portfolioId/investments

Get all investments for a portfolio

POST /portfolios/:portfolioId/investments

Add new investment to portfolio

GET /portfolios/:portfolioId/investments/:investmentId/performance

Get investment performance metrics and analytics

Dashboard & Analytics Endpoints

GET /dashboard/summary

Get overall dashboard summary with key metrics

GET /dashboard/portfolio/:portfolioId/summary

Get detailed summary for specific portfolio

Get asset allocation breakdown and distribution

API Features

  • Authentication: JWT-based with access and refresh token mechanism
  • Authorization: Protected routes require valid access token
  • Validation: Request body validation on all POST/PUT endpoints
  • Error Handling: Consistent error response format across all endpoints
  • Soft Deletes: Portfolio deletions are soft deletes (recoverable)
